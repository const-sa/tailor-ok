# دليل ربط الفوترة الإلكترونية — المرحلة الثانية (هيئة الزكاة / فاتورة)

نظام: SewingSystem (ثوبي الفاخر) — فواتير **مبسطة B2C** عبر **Reporting** — البيئة الحالية: **محاكاة (Simulation)**.

---

## ١) ما تم تنفيذه حتى الآن (الأساس)

### قاعدة البيانات
- **حقول جديدة في `tblSellInvoice`** لتتبّع المرحلة الثانية:
  `ZatcaUUID, ZatcaICV, ZatcaPIH, ZatcaInvoiceHash, ZatcaQR, ZatcaTypeCode, ZatcaStatus, ZatcaResponse, ZatcaReportedAt, IsReturn, OriginalInvoiceID`.
- **جدول إعدادات `tblZatcaConfig`** (صف واحد، `Id=1`): البيئة، بيانات المنشأة، العنوان، المفتاح الخاص والشهادات (مشفّرة)، وآخر `ICV/PIH`.
- **بيانات بائع افتراضية + تفعيل الضريبة 15%** على الفرع (`tblBranche`).

### الكود (`Classes/Zatca/`)
| الملف | الوظيفة |
|---|---|
| `ZatcaConfig.cs` | قراءة/حفظ الإعدادات، تحديد روابط البيئة وقالب الشهادة |
| `ZatcaCrypto.cs` | توليد مفتاح `secp256k1` + طلب الشهادة CSR بقالب الزكاة |
| `ZatcaQr.cs` | بناء QR المرحلة الثانية (الوسوم ١–٩) وتوليد صورته |

> ⚠️ القيم الحالية **افتراضية صالحة الشكل فقط** — استبدلها ببياناتك الحقيقية قبل الإنتاج.

## ٢) البيانات الافتراضية الحالية (غيّرها لاحقاً)

| الحقل | القيمة الحالية |
|---|---|
| الرقم الضريبي (VAT) | `300000000000003` |
| الاسم النظامي | ثوبي الفاخر |
| السجل التجاري (CR) | `1010010000` |
| الرقم التسلسلي للجهاز (EGS) | `1-TailrMdinah\|2-POS01\|3-100000000000001` |
| العنوان | الرياض — حي العليا — الملك فهد — 12211 |
| البيئة | `simulation` |

**لتغييرها** عبر SQL (مثال):
```sql
UPDATE dbo.tblZatcaConfig SET
  VatNumber='3XXXXXXXXXXXXX3', OrgName=N'الاسم النظامي', CrNumber='10xxxxxxxx',
  AddrStreet=N'اسم الشارع', AddrBuilding='1234', AddrDistrict=N'الحي',
  AddrCity=N'المدينة', AddrPostal='12345', Environment='simulation'
WHERE Id=1;
```

## ٣) ما تبقّى لإكمال التنفيذ (المراحل القادمة)

### المرحلة ٢ — التهيئة والربط الفعلي (تحتاج إنترنت + OTP)
1. **`ZatcaApiClient.cs`** — عميل HTTP لنقاط الهيئة:
   - `POST /compliance` (طلب CCSID بـ CSR + OTP)
   - `POST /compliance/invoices` (فحوصات الامتثال)
   - `POST /production/csids` (طلب PCSID)
   - `POST /invoices/reporting/single` (إبلاغ الفواتير المبسطة)
2. **`UblInvoiceBuilder.cs`** — توليد فاتورة XML بصيغة UBL 2.1 (مبسطة + إشعار دائن).
3. **`ZatcaSigner.cs`** — حساب الهاش (SHA256) + التوقيع الرقمي XAdES + إكمال وسوم QR (٦–٩).
4. **`ZatcaService.cs`** — تنسيق العملية: تهيئة → توقيع → إبلاغ → حفظ الحالة وتسلسل الهاش.

### المرحلة ٣ — شاشة الإعدادات + زر التهيئة
- شاشة «إعدادات ربط الزكاة»: تعديل البيانات، اختيار البيئة، إدخال OTP، أزرار:
  «توليد المفاتيح/CSR» → «طلب شهادة الامتثال» → «فحوصات الامتثال» → «طلب شهادة الإنتاج».

### المرحلة ٤ — شاشة المرتجع (الإشعار الدائن)
- اختيار فاتورة أصلية → اختيار الأصناف/الكميات (كامل أو جزئي) → إنشاء مرتجع نوع `381`
  يعكس الضريبة ويُرجع المخزون، ثم توقيعه وإبلاغه للهيئة.

### المرحلة ٥ — الطباعة
- إضافة QR المرحلة الثانية للفاتورة المطبوعة، وعرض حالة الإبلاغ.

---

## ٤) خطوات التهيئة التي ستقوم بها أنت (عند الوصول للمرحلة ٢)

1. ادخل **بوابة فاتورة (بيئة المحاكاة)**: <https://fatoora.zatca.gov.sa> → سجّل دخول بنفاذ.
2. من «الأجهزة/EGS» أنشئ وحدة جهاز جديدة واطلب **رمز OTP** (صالح لدقائق فقط).
3. في النظام: افتح «إعدادات ربط الزكاة» → الصق الـ OTP → اضغط **«توليد + طلب شهادة الامتثال»**.
4. شغّل **فحوصات الامتثال** (يرسل النظام عينات: فاتورة مبسطة + إشعار دائن).
5. عند النجاح اضغط **«طلب شهادة الإنتاج»** → يصبح النظام جاهزاً للإبلاغ التلقائي.
6. للانتقال للإنتاج لاحقاً: غيّر `Environment='production'` وكرّر التهيئة ببيانات الإنتاج.

---

## ٥) متطلبات الإنتاج (لاحقاً)
- رقم ضريبي حقيقي مسجّل (15 رقماً يبدأ وينتهي بـ3) + سجل تجاري + عنوان وطني صحيح.
- اتصال إنترنت من جهاز الكاشير بنقاط الهيئة.
- نسخ احتياطي للمفتاح الخاص والشهادات (مخزّنة مشفّرة في `tblZatcaConfig`).

---

## ملاحظات تقنية
- التوقيع يعتمد منحنى `secp256k1` ومكتبة **BouncyCastle** (موجودة في المشروع).
- QR يُبنى بمكتبة **ZXing** (موجودة).
- الأسرار (المفتاح الخاص/أسرار CSID) تُخزَّن مشفّرة بنفس مفتاح AES المستخدم في النظام.
- تسلسل الفواتير: كل فاتورة تحمل `ICV` متزايداً و`PIH` = هاش الفاتورة السابقة (الأولى تبدأ بهاش التكوين Genesis).
